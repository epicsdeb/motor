Installation and Building

After obtaining a copy of the distribution, it must be installed and built for use at your site. These steps only need to be performed once per site (unless versions of the module running under different releases of EPICS and/or the other required modules are needed).
  1. Create an installation directory for the module, usually this will end with 
    2. .../support/motor/
  2. Place the distribution file in this directory. Then issue the commands (Unix style) 
    3. gunzip motor<rel>.tar.gz
    tar xvf motor<rel>.tar
    where <rel> is the release. For example. 
    motor4-1.tar.gz
  3. This creates a <top> application. 
    4. .../support/motor/motor<rel>
  4. Edit the config/RELEASE file and set the paths to your installation of EPICS base (i.e., define SUPPORT and EPICS_BASE).
  5. Edit  .../support/motor/motor<rel>/motorApp/Makefile.  Define which device/driver modules to build.
  6. Run gnumake in the top level directory and check for any compilation errors.
